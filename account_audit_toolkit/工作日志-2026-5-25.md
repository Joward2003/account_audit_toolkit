## 2. 构建 Evidence Checker 完整流水线（evidence_checker.py）

### 2.1 EvidenceLoader
- 从 supporting Excel（如 支持性附件截图.xlsx）按 sheet 提取嵌入图片
- 通过 zipfile + drawing rels 做 sheet → drawing → image 精确映射
- 支持 `SHEET_TO_NODE` / `FULL_NODE_TO_SHEET` 字典进行 node-sheet 路由

### 2.2 OCRExtractor
- 封装 RapidOCR（rapidocr_onnxruntime），输出带像素级 bbox 的 TextBox 列表
- 每张图片输出 OCRPage {text_blocks[], full_text, image_index}

### 2.3 EvidenceChecker（三层比对策略）

**第一道：表头空间锚定**
- 根据 ANCHOR_KEYWORDS 字典找到对应节点的锚点关键词
- 自动推断列边界（列间隙检测）
- 提取列内文本并按 Y 坐标同行合并
- 不同节点有独立的锚点关键词配置

**第二道：MySQL 特化归一化 + 全量文本池兜底**
- `normalize_account_mysql()`: 修复 RapidOCR 对终端字体的常见误读（%→*, @→Q 等）
- 对数据库节点自动启用 MySQL 归一化
- 全量文本池精确 + 子串包含匹配

**第三道：字符级模糊匹配**
- `fuzzy_char_ratio()`: Jaccard + 长度惩罚的字符重叠率算法
- 阈值 0.75，用于兜底 OCR 字符级误读（如 dba@% 被读成 dlba）

### 2.4 集成回 execution_plan
- `run_evidence_check()` 主流程：读取 evidence_plan → 加载截图 → OCR → 比对 → 写回 compare_result
- 支持 `--system` 过滤，只检查指定系统的节点
- compare_result 的 ActualAccounts / MissingAccounts / CheckResult / Remark 字段自动覆写

## 4. 关键发现

- 表头空间锚定对 Jenkins UI 类页面命中率 100%（9/9 账号全部锚定命中）
- MySQL 终端截图（小字体 monospace）是 RapidOCR 的弱点：特殊字符 `@` `%` 易被误读
- MySQL 归一化（*→%）修复了大部分字符误读，将匹配率从 4/8 提升到 7/8
- REPL@% 是仅剩的 OCR 硬伤（读成 rep10*/repl0s），字符损坏严重，标记 REVIEW 人工复核
