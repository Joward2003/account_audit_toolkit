## 2. main.py 重构为端到端审阅结论生成器

### 改动前
`main.py` 只做 Scope Engine → 导出 todo_list / program_template / JSON。

### 改动后
完整串联 5 个阶段：

```
WP Excel (test.xlsx)
  → Scope Engine (rules_scope.py)
  → System-Folder Mapper
  → Execution Plan Builder (build_expected_accounts + decide_evidence_plan)
  → Evidence Checker (OCR 截图比对)
  → account_audit_conclusion.xlsx（最终审阅结论）
```

### 核心设计原则：复用而非重写

直接 import `execution_plan_builder` 和 `evidence_checker` 的关键函数：

| 复用的函数 | 作用 |
|---|---|
| `build_expected_accounts_for_task()` | 从账号审阅导出构建 expected accounts，返回 ManualUploadOverall（Y/N） |
| `decide_evidence_plan()` | 判断该节点怎么检查：ParseMethod（export_available / no_manual_upload / account_presence_or_matrix_review） |
| `EvidenceLoader.load_for_node()` | 从 supporting Excel 按 sheet 提取嵌入图片 |
| `OCRExtractor.extract()` | RapidOCR 文本检测 |
| `EvidenceChecker.check()` | 表头锚定 + 全量兜底双保险比对 |

### 测试逻辑（来自 execution_plan_builder）

```
手工上传标识 = N → 只需账号存在即通过 → 「是」
手工上传标识 = Y → 账号存在 + 截图 OCR 通过 → 「是」
```

仅 `ParseMethod` 属于 OCR 需要型（account_presence_or_matrix_review 等）且 `ManualUploadOverall == "Y"` 时才跑 OCR。

### 派生检查
- 需上传开发人员清单：文件存在即通过
- 开发人员权限：在审阅数据中查找，不允许 读写/管理员
- SOD：应用层管理员 vs 数据库/服务器管理员无重合

### 运行结果

| 系统 | 通过 | 结论 |
|---|---|---|
| **OPM** | 7/10 | 3项Y但无supporting Excel → 否-截图检查未通过；7项N → 通过 |
| **YUMC Zoo** | 10/10 ✅ | 全部通过（OCR Y项全部匹配，N项账号存在，派生检查通过） |

### 输出文件
`output/account_audit_conclusion.xlsx`
- Sheet「审阅结论」：WP 原列 + 10 个审计节点结论（是/N/A/否-xxx）+ 通过率
- Sheet「Scope明细」：Pipeline 完整明细
